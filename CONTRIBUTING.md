## Issues and Feature Requests

If you encounter a bug or have an idea for a new feature, please first check the existing [issues](https://github.com/xakpc/DotnetPrompt/issues) to see if it has already been reported. If not, feel free to create a new issue, describing the problem or feature in as much detail as possible.

## Pull Requests

We are happy to accept pull requests from anyone. Before submitting a pull request, please ensure that:

- The code is accompanied by appropriate tests.
- The code follows the existing code style and adheres to any relevant coding standards.
- The commit messages are descriptive and follow the [conventional commit](https://www.conventionalcommits.org/en/v1.0.0/) format.
- The pull request includes a brief summary of the changes and any relevant context.

## Getting Started

To get started contributing, follow these steps:

- Fork the repository and clone it locally.
- Create a new branch for your changes.
- Make your changes and ensure that they are accompanied by appropriate tests.
- Commit your changes using the [conventional commit](https://www.conventionalcommits.org/en/v1.0.0/) format.
- Push your changes to your forked repository.
- Submit a pull request to the main repository's `main` branch.

## Code of Conduct

Please note that this project follows the Contributor Covenant Code of Conduct. By participating in this project, you are expected to uphold this code. If you encounter any violations, please contact the project maintainers.
